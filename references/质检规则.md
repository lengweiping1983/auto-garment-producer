# 质检规则

纹理质检是可靠性闸门。OpenCV 渲染只能使用已批准的纹理资产。

## 自动检查

基础版 `质检纹理.py` 检查以下内容：

- 文件存在且可打开
- 图像尺寸足够大
- 纹理具有足够的视觉变化
- 对边足够相似以保证平铺
- 存在 approved 标志

## 必需的人工或 AI 检查

还需检查：

- 无动物
- 无人物/人脸
- 无文字/商标/水印
- 无强烈中心构图
- 无可见边框
- 色板与主题匹配
- 纹理密度与分配的裁片兼容
- 大身裁片比点缀裁片更低噪
- 窄条/袖口不过度堆砌
- 卖点图案出现在不超过 1–2 个关键裁片上
- 成对裁片协调或有意镜像

## 审批闸门

仅允许具有以下状态的面料通过：

```json
{
  "approved": true,
  "qc": {
    "approved": true,
    "tileable_score": 0.75
  }
}
```

如果质检失败：

- 重新生成面料
- 或将其修复为可平铺图块
- 或用纯色替换

不得将失败面料推入生产渲染。

## 商业时尚质检

渲染后还应创建 `成品质检报告.json`，标记以下问题：

- 卖点裁片过多
- 所有裁片使用相同纹理或相同密度
- 饰边/细条裁片使用复杂图案
- `裁片填充计划.json` 中缺少 reason
- 引用未批准的图案
- 渲染后的 PNG 没有透明 Alpha
- 色板偏离已批准的商业范围

## 商业复审两轮制

最终预览必须以“一件已经缝合的衣服”复审，而不是以单独裁片复审。

第 1 轮：标准商业复审。

- 输入：Kimi 压缩预览图、`piece_fill_plan.json`、`commercial_design_brief.json`、可选 `fashion_qc_report.json`。
- 输出：`ai_commercial_review.json`。
- 任一硬否决条件触发时，必须 `approved=false`，无视 AI 自评分数。

硬否决条件：

- 三米测试不通过：远看主体不清、纹理糊、整体花乱、像贴图拼接。
- Hero Motif 几何契约越界：过大、靠边、跨缝线、超过 1 个 hero、半透明整图 overlay。
- 双源资产跨源数量超过 2 个小面积 accent/trim，或大身 base 混用 A/B 两源。
- `scores.hero_clarity` 或 `scores.style_cohesion` 低于 8。
- 大身 base 出现完整动物、人脸、文字、完整场景、建筑、复杂叙事插画。
- 出现半透明矩形残影、未去干净背景、明显水印或贴片边界。

第 2 轮：仅当第 1 轮 `approved=false` 时执行 fallback。

- 指令必须要求重选资产和重做填充计划，而不是只调整缩放或偏移。
- 输出 `fallback_safe_plan`：放弃 hero motif；大身只用低噪 base 或纯色；袖片与身片同源；trim/collar/cuff 使用最低饱和度 secondary 或 solid。
- fallback 版本不追求视觉冲击，目标是统一、可穿、可量产。

交付优先级：

1. 第 1 轮通过的方案。
2. 第 2 轮 fallback safe plan。
3. 若 fallback 仍不通过，终止流程并报告失败原因。

未通过 v1/v2 的方案不得交付，也不得仅凭局部修图进入量产渲染。
